              PHASE 10: MULTI-TENANT FOUNDATION (TENANT = COMPANY)
================================================================================

Status: PLANNED (Not yet implemented)
Last Updated: January 2026

================================================================================
OVERVIEW
================================================================================

Implement multi-tenant data isolation where Tenant = Company.

Data Model:
```
User <--(many-to-many)--> Organization <--(one-to-many)--> Company (= Tenant)
                                                              |
                                                              +-- Vendors
                                                              +-- Bills
                                                              +-- Projects
                                                              +-- Customers
                                                              +-- ... (all transactional data)
```

================================================================================
NAMING CONVENTION (HYBRID APPROACH)
================================================================================

Layer                    Column/Field Name
----------------------   -----------------
JWT payload              company_id
Entity tables (SQL)      CompanyId
Python models            company_id
TriggerContext           tenant_id (maps from company_id)
Workflow table           TenantId (unchanged)

Rationale:
- Domain code uses clear, entity-specific naming (company_id)
- Workflow infrastructure keeps abstract naming (tenant_id)
- Single source of truth: company_id in JWT, mapped where needed

SUB-PHASE 10a: DATABASE SCHEMA CHANGES
================================================================================

Scope: Create junction tables, add OrganizationId to Company, add CompanyId 
       to transactional tables

Tasks:
  1. Create UserOrganization junction table
     - UserId (FK to User)
     - OrganizationId (FK to Organization)
     - Role (owner, admin, member)
     - Unique constraint on (UserId, OrganizationId)

  2. Add OrganizationId FK to Company table
     - ALTER TABLE dbo.Company ADD OrganizationId BIGINT
     - Add foreign key constraint to Organization

  3. Add CompanyId FK to transactional tables (nullable initially)
     Tables to update:
     - Vendor
     - Bill
     - Project
     - Customer
     - CostCode
     - SubCostCode
     - PaymentTerm
     - ContractLabor
     - VendorType
     - Taxpayer

  4. Create indexes for new CompanyId columns

  5. Update stored procedures to accept/filter by CompanyId

Files to Create:

Files to Modify:

SUB-PHASE 10c: AUTH & JWT CHANGES
================================================================================

     - Remove hardcoded tenant_id: 1

SUB-PHASE 10e: DATA MIGRATION & BACKFILL
================================================================================

Scope: Migrate existing data to new schema

Tasks:
  1. Create default organization if none exists
     - Name: "Default Organization" or use first existing

  2. Link existing companies to default organization
     - UPDATE Company SET OrganizationId = @DefaultOrgId

  3. Link existing users to default organization
     - INSERT INTO UserOrganization for each user

  4. Backfill CompanyId on all transactional tables
     - Use default company (ID=1) for existing records
     - UPDATE Vendor SET CompanyId = 1 WHERE CompanyId IS NULL

  5. Add NOT NULL constraints after backfill
     - ALTER TABLE Vendor ALTER COLUMN CompanyId BIGINT NOT NULL

  6. Verify data integrity
     - Check all records have valid CompanyId
     - Verify foreign key relationships

Files to Create:
  - scripts/migrate_multi_tenant.py (or SQL migration script)
